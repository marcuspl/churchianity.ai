## Lens 3 — Licensing, monetisation, and self-publishing

### Convergence (all three agents)
- **Dual model is viable**: free web + paid print/ebook.
- **Avoid KDP Select** if the web version remains free, because Select requires **ebook exclusivity** (KDP says ebooks in Select must be exclusive to the Kindle Store for the enrollment period). See KDP help: [How to enroll in KDP Select](https://kdp.amazon.com/en_US/help/topic/GD9PMU58BV24QFZ7).
- **POD shortlist**: KDP Print + IngramSpark (plus Lulu as an optional direct-fulfillment fallback).

### Key disagreement: CC license variant (ND vs SA vs “just NC”)
- **Gemini:** prefers **CC BY-NC-ND** (“safest” integrity protection; blocks derivatives).
- **Claude:** prefers **CC BY-NC-SA** (allows derivatives but forces same license downstream).
- **Me:** I leaned **CC BY-NC** as the simplest expression of “share freely, don’t resell,” while warning that **NC can block at-cost church bookstores** unless you explicitly grant permission.

**My synthesis recommendation:** If the goal includes *translations, study guides, or remixes*, prefer **CC BY-NC-SA**; if the goal is *maximum text integrity control*, prefer **CC BY-NC-ND**. Either way, add an explicit permission statement if you want churches/educators to print at cost (NC’s edge case).

Reference for CC BY-NC terms: [CC BY-NC 4.0 deed](https://creativecommons.org/licenses/by-nc/4.0/).

### Addition: KDP pricing dynamics / “free ebook” realities
Gemini usefully notes Amazon’s **price matching** behavior: if the ebook is free elsewhere (Apple/Google/Kobo or direct download), Amazon may match to $0.00. This can be either a feature or a headache depending on whether you want Kindle to be free.

Also: KDP’s own price requirements matter if you want 70% royalty for paid Kindle pricing (KDP list price rules differ by territory and file size). See: [KDP eBook list price requirements](https://kdp.amazon.com/en_US/help/topic/G200634560).

### Addition: “what NOT to do” list is aligned
All three of us converge on practical pitfalls:
- don’t skip print proofs (footnotes, special characters, italics, and section breaks are where POD breaks),
- don’t ship without a professional cover,
- don’t surprise readers with AI disclosure mismatches (keep the same transparency posture on store listings).
